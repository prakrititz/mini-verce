Deployment engine

Deployments are built one at a time, recorded in SQLite, and routed through Caddy after the new container starts.

Build flow

  1. Detect the framework and read minivercel.json when present.
  2. Generate a missing Dockerfile and .dockerignore for Next.js, Vite, CRA, or plain Node apps.
  3. Load project env vars and pass them as Docker build args and container env entries.
  4. Pick a free host port and build the image with dockerode.
  5. Start the container with host port bindings for 80, 3000, and 8080.
  6. Insert a running deployment row, regenerate the Caddyfile, and reload Caddy.

Cleanup behavior

  • Production deploys stop older running production containers after the new one is live.
  • Preview deploys are keyed by PR-specific URLs and are cleaned up independently.
  • The deployer retains up to three older deployments for rollback and removes older stopped containers.

URLs

  • Production routes use http://<project-name>.localhost:8080.
  • Preview routes use http://pr-<number>.<project-name>.localhost:8080.
