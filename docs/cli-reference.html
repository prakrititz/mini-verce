CLI reference

The CLI mirrors the current daemon surface: auth, project ownership, GitHub linking, Caddy mode controls, deploys, logs, and rollback.

Daemon control

  • start-daemon starts the background daemon and Caddy container.
  • caddy trust installs the local CA for HTTPS in local mode.
  • caddy mode prints whether Caddy is running in local or public TLS mode.

Auth

  • signup creates a local account and session.
  • login starts a new session for an existing account.
  • logout invalidates the session and removes the auth file.
  • whoami reads the cached auth file.

GitHub

  • github connect stores a PAT for repo listing and webhook registration.
  • github status shows whether a GitHub account is connected.
  • github disconnect clears stored GitHub credentials.
  • github oauth uses the OAuth callback flow when GITHUB_CLIENT_ID and GITHUB_CLIENT_SECRET are configured.
  • import lists repos from the connected GitHub account and links the selected repo.

Project actions

  • link registers the current folder as a project owned by the logged-in user.
  • deploy queues a production deployment for the linked path.
  • list shows owned projects and their latest deployment state.

Operations

  • env add/rm/pull manages variables for the current project.
  • logs streams the running container logs, optionally following them.
  • domain add/rm updates the routed custom domain.
  • rollback returns to a recently stopped production deployment.
