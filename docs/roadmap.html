Roadmap

The old pipeline documents are still useful historically, but this branch already implements many of the V2 goals they described.

Already implemented

  • signup, login, logout, and session-backed auth
  • project ownership and protected project routes
  • GitHub PAT or OAuth linking plus webhook registration
  • local TLS mode and public TLS mode in Caddy
  • preview deployments and rollback to retained deployments

Still planned

  • A dashboard at /dashboard.
  • Resource limits and build cache reuse.
  • Persistent queue state and richer lifecycle commands like status, stop, and restart.
  • Multi-domain project modeling and plugin-style framework adapters.
  • File-watch dev mode.

Current caveats

  • Environment values are stored as plaintext.
  • Rollback relies on retained containers.
  • The queue is in-memory and disappears on daemon restart.
