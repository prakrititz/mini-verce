Database

Mini-Vercel stores its state in SQLite. The current schema includes users, sessions, projects, deployments, env vars, and stored GitHub credentials.

Main tables

  • users stores local accounts plus github_username and encrypted PAT/OAuth token columns.
  • sessions stores login tokens with expiry and is cleaned on startup.
  • projects stores name, path, repository URL, custom domain, and owner_id.
  • deployments stores status, container id, port, env, URL, and timestamps.
  • env_vars stores project variables keyed by name.

Important notes

  • Project names are globally unique in the current schema.
  • Environment values are stored as plaintext.
  • Production and preview deployments share the same table, distinguished by the env and url columns.
  • Legacy rows are migrated with safe ALTER TABLE attempts on startup.
