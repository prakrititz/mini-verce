Operations

The platform keeps a single-build queue, validates GitHub webhooks, and prunes stale data on a schedule.

Queue and logs

  • The build queue runs one job at a time and reports queue length plus busy state through /health and /api/queue.
  • mini-vercel logs reads the running container directly from Docker and tails the last 100 lines, with --follow support.

Webhooks

  • GitHub webhooks are limited to 10 requests per minute.
  • When WEBHOOK_SECRET is set, the daemon verifies x-hub-signature-256 with HMAC SHA-256.
  • Push events on main or master enqueue production deployments.
  • Pull request events create, update, or tear down preview deployments.

Maintenance

  • The daemon runs docker system prune -f every hour.
  • Expired sessions are removed on the same interval.
  • Stopped production deployments older than seven days are deleted.
