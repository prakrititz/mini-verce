Caddy routing

Caddy is the local reverse proxy. The branch now supports local TLS with Caddy's internal CA as well as a public mode for real domains.

Modes

  • local mode writes { local_certs } at the top of the Caddyfile and appends tls internal to each site block.
  • public mode leaves TLS to Caddy's automatic certificate flow.
  • mini-vercel caddy trust installs the local CA once so https://<project>.localhost works cleanly.
  • mini-vercel caddy mode reports the selected mode and ports.

Routing model

  • Production projects route at https://<project>.localhost in local mode and the bare hostname in public mode.
  • Custom domains use the same running container and are emitted as additional site blocks.
  • Preview deployments route at pr-<number>.<project>.localhost.
  • Each site block reverse proxies to host.docker.internal:<port>.

Reload behavior

The daemon rewrites Caddyfile from the current running deployments and then runs caddy reload --config /etc/caddy/Caddyfile --force inside the container.
