Core workflow

The current branch is centered on local identity, project ownership, and a daemon-driven deploy loop.

  1. Start Docker, then run mini-vercel start-daemon.
  2. Create an account with mini-vercel signup or authenticate with mini-vercel login.
  3. Link the current directory with mini-vercel link or import a GitHub repo with mini-vercel import.
  4. Add environment variables if the app needs them.
  5. Run mini-vercel deploy or trigger a webhook-backed deploy from GitHub.
  6. Visit the routed local hostname, or use preview URLs for pull requests.

What happens on deploy

  • The daemon resolves the project by path and enforces ownership.
  • Framework detection writes missing Docker assets if needed.
  • Environment variables are loaded from SQLite and passed into the build and container.
  • The image is built, a free host port is selected, and the container is started.
  • Caddy is regenerated and reloaded so traffic flips to the new container.

GitHub flow

  • mini-vercel github connect stores a PAT for repo listing and webhook registration.
  • mini-vercel github oauth uses the daemon's OAuth callback when configured.
  • Pushes to main or master enqueue production deploys.
  • Pull request events create or tear down preview deployments.
