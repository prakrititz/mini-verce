Daemon API

The Express daemon owns auth, project ownership, GitHub integration, webhooks, Caddy control, and deploy execution.

Public routes

  • GET /health returns daemon and queue status.
  • GET /api/queue reports queue length and busy state.
  • GET /api/caddy/mode returns local or public TLS mode and the exposed ports.
  • POST /api/caddy/trust runs caddy trust inside the container.
  • POST /auth/signup, /auth/login, and /auth/logout handle sessions.
  • POST /webhooks/github validates GitHub webhooks with HMAC when WEBHOOK_SECRET is set.

Auth and GitHub routes

  • POST /auth/github/connect validates a PAT and stores encrypted GitHub credentials.
  • GET /auth/github/status reports whether GitHub is connected.
  • DELETE /auth/github/disconnect clears stored GitHub credentials.
  • GET /auth/github/start begins the OAuth redirect flow when configured.
  • GET /auth/github/callback completes the OAuth exchange and stores the encrypted token.
  • GET /api/github/repos lists repos from the connected GitHub account.
  • POST /api/projects/link-github links a selected repo and registers a webhook.

Protected project routes

  • POST /api/projects/link links the current directory to the logged-in user.
  • GET /api/projects lists only owned projects.
  • GET /api/projects/by-path resolves the current path for the current user.
  • GET /api/projects/:id returns one owned project.
  • POST /api/projects/:id/deploy enqueues a production deployment.
  • POST /api/projects/:id/rollback flips status back to a previous production deployment.
  • PATCH /api/projects/:id/domain sets or clears a custom domain.
  • GET /api/projects/:id/env, PUT /api/projects/:id/env, and DELETE /api/projects/:id/env/:key manage environment variables.
